# 財務・人事部門へのデータ依頼ガイド

## 【推奨】いちばん簡単な様式：roster_form.csv（空欄記入式）

**`roster_form.csv` を送って「人数」欄を埋めてもらうだけ**の様式を用意しました。
雇用形態×性別×年齢階級（5歳刻み）の**100行が事前印字**されており、先方の作業は
該当行に在籍人数を入れるだけです（該当なしの行は空欄のまま）。

- 「平均勤続年数」「平均年収」列は**任意**（空欄なら全国データの同区分中央値で
  自動補完されるため、埋まらなくても分析可能）
- Excel でそのまま開けます（UTF-8 BOM付き）。記入例は `roster_form_example.csv`
- 返送されたファイルは、物価等の加工を**一切せずそのまま** Colab ノートブック
  （`workplace_attrition_colab.ipynb`）で読み込めます（マクロ情報は自動付与）

以下の詳細版（多次元ピボット）は、先方に余力がある場合の高精度版です。


## 【重要】means_form.csv（15マスの必須ミニ表）

セル別の「平均勤続」「平均年収」列は空欄で返ってくることが多いため、代わりに
**雇用形態別の平均だけを別紙 `means_form.csv`（3職種グループ×5雇用形態=15マス）
で必須**としてお願いする設計です。給与システムの集計で数分で出せる粒度で、
「概算で可」。この15マスがあると、全国データの年齢勾配を職場の水準に合わせて
スケーリングでき、推計バイアス（検証では約0.8〜0.9%ポイント）を大幅に削減
できます。該当者がいない雇用形態のマスは空欄で構いません。

## 依頼のポイント（事務方の負担を最小にする設計）

給与システム・人事システムには通常「氏名・生年月・採用年月・雇用区分・支給額」が
そろっているので、**ピボット集計（度数表）1枚**を出してもらう形が最も対応しやすい
です。個票（個人別データ）は求めません。

- 求めるのは**区分別の在籍人数のみ**（氏名・職員番号・個別金額は一切不要）
- 区分（バンド）は粗くて構わない旨を明記する（先方の裁量余地を残すと早い）
- 提出form: 同梱の `roster_template.csv` に人数を記入（Excelでも可）

## 依頼文の例（そのまま使えます）

> 【依頼】事務系職員の構成に関する集計表の作成について
>
> 人員構成の中期見通しに関する統計分析（内部検討資料）のため、
> ○年○月○日時点で在籍する事務系職員について、以下の区分別の
> **在籍人数の集計表**を作成いただけないでしょうか。
>
> - 区分：雇用形態（正規／契約／パート／派遣／嘱託）× 性別 ×
>   年齢階級（5歳刻み）× 勤続年数階級（0–4 / 5–9 / 10–19 / 20–29 / 30以上）×
>   年収階級（100万円刻み、税込支給ベースの概算で可）
> - 必要なのは**各区分の人数のみ**です。氏名・職員番号など個人を特定する
>   情報や、個人別の金額は一切不要です。
> - 区分が細かく作成が難しい場合は、年収階級を3区分（〜300／300–500／500以上）
>   にする等、粗い区分で構いません。該当者が1名となるセルの扱いに懸念が
>   ある場合はご相談ください（近隣セルとの統合で対応可能です）。
> - 様式：添付のCSV（Excel可）の headcount 欄に人数をご記入ください。
>
> お手数をおかけしますが、よろしくお願いいたします。

## 様式の説明（roster_template.csv）

| 列 | 内容 | 記入例 |
|---|---|---|
| employment_type | 正規/契約/パート/派遣/嘱託 | 正規 |
| sex | 男/女/不明 | 女 |
| age_band | 年齢階級 "下限-上限" | 45-49 |
| tenure_band | 勤続年数階級 "下限-上限" | 10-19 |
| salary_band_manen | 年収階級（万円）"下限-上限" | 500-599 |
| headcount | その区分の人数 | 5 |

- 行は必要なだけ追加可。バンド幅は行ごとに違ってよい（例: "30-39" と "40-44" が混在して可）。
- 記入例は `roster_example.csv`（架空の約80人の事務組織）。

## 取得できなくても問題ない項目

以下は給与台帳に通常なく、**無くても分析可能**です（モデルが欠測をそのまま
扱える設計のため）：学歴、配偶者・子どもの有無、週労働時間、役職。
もし雇用形態別の役職者数など追加で出せるなら精度は上がりますが、必須では
ありません。

## 個人情報の取り扱い

- 受け取るのは統計表（集計値）のみで、個人情報には該当しない水準です。
- 分析はすべて手元（ローカルPC / 自分のGoogle Drive）で完結し、外部
  サービスへ送信しません（`workplace_attrition.ipynb` はローカルで動作）。
- それでも所属機関の内規（目的外利用・内部資料の扱い）に沿った手続きを
  推奨します。
